# 产出材料目录规范

本规则强制约束所有技能产出材料的存放位置与命名，保障项目结构清晰。**任何产出文件不得散落在项目根目录或随意放置。**

## 1. 目录映射表

| 材料类型 | 存放目录 | 文件格式 | 对应技能 |
|----------|----------|----------|----------|
| PPT 演示文稿 | `ppt/` | `.html` + `.pptx` | po-05-frontend-slides-ppt + dom-to-pptx |
| 流程图/架构图/时序图 | `diagram/` | `.svg` / `.drawio` / `.md` | po-02-baoyu-diagram-流程图 |
| 页面原型 | `prototype/` | `.html` | po-04 / po-07 / po-09 |
| 需求文档（PRD） | `prd/` | `.md` | po-01-to-prd-需求文档 |
| 日报/周报 | `report/` | `.md` | pm-01-daily-report-日周报 |
| 信息图/模块图 | `infographic/` | `.svg` / `.html` | po-03-baoyu-infographic-模块图 |
| 临时文件 | `temp/` | 任意 | po-06-brainstorming 等 |

## 2. 输入材料目录

| 目录 | 用途 | 说明 |
|------|------|------|
| `docs/` | 原始需求文档 | 用户上传的需求、立项材料、参考资料；也存放教程文档 |
| `design-system/` | UI/UE 规范 | 原始规范文件或总结提炼的规范（配色、字体、组件、PPT 设计规范） |

## 3. 命名规范

### 3.1 通用规则

- 使用**中文或英文短语**命名，语义清晰，避免无意义缩写
- 文件名中**不要包含空格**，用 `-` 或 `_` 连接
- 避免特殊字符：`/ \ : * ? " < > |`
- 同一主题的多个文件用**子目录**组织，不要平铺

### 3.2 子目录组织

每个产出目录下，按「项目/主题」建立子目录聚合相关文件：

```
ppt/
├── 矿产平台安全汇报/
│   ├── 矿产平台安全汇报.html
│   └── 矿产平台安全汇报.pptx
└── 2026半年会/
    └── ...

diagram/
├── backend-dev-process/
│   ├── flowchart.svg
│   ├── flowchart.drawio
│   └── 后端研发流程说明.md
└── ...

prototype/
├── 矿产平台首页/
│   └── index.html
└── ...
```

### 3.3 版本管理

- 需保留多版本时，文件名追加日期后缀：`矿产平台安全汇报-20260709.html`
- **不要**用 `final`、`v2`、`最新` 等模糊后缀，统一用日期 `YYYYMMDD`

## 4. 临时文件管理

- 所有过程性、试验性、可丢弃的文件**统一放入 `temp/`**
- 包括但不限于：草稿、调试用的 HTML、下载的素材、中间产物
- `temp/` 目录**可随时整体清理**，不保留任何正式产出
- 正式产出确认后，必须从 `temp/` 移动到对应产出目录

## 5. PPT 产出特别约定

- HTML 源文件与导出的 `.pptx` **成对存放**在同一子目录
- HTML 文件命名与 `.pptx` 保持一致（仅扩展名不同）
- 不得将 HTML 与 PPTX 分散到不同目录

## 6. 禁止行为

| 禁止 | 应当 |
|------|------|
| 在项目根目录直接产出文件 | 放入对应类型目录 |
| 把临时文件放在 `ppt/`、`diagram/` 等正式目录 | 放入 `temp/` |
| 用 `最终版.pptx`、`最新.html` 命名 | 用 `名称-YYYYMMDD.pptx` |
| 多主题文件平铺在一个产出目录 | 按主题建子目录 |
| 修改 `design-system/` 中的规范而不记录 | 规范变更需在规范文件中注明更新日期 |

## 7. 新增技能时的目录登记

新增技能后，需同步更新两处：
1. `README.md` 的「可扩展技能体系」表格——登记技能名、用途、产出目录
2. 本文件第 1 节「目录映射表」——登记材料类型与存放目录

## 8. 目录初始化清单

新建产出目录时，须放置 `.gitkeep` 文件以保留目录结构（已被 `.gitignore` 排除内容，仅保留结构占位）。

```
ppt/.gitkeep
diagram/.gitkeep      # 已有内容，无需占位
prototype/.gitkeep
prd/.gitkeep
report/.gitkeep
infographic/.gitkeep
temp/.gitkeep
```
